Build a real-time chat/messaging system for CreatorTasks inspired by Fiverr/Upwork, focused on trust, clarity, and preventing abuse.

Goal:
Enable communication between task creators and workers while keeping conversations safe, structured, and tied to tasks.

----------------------------------
Backend
----------------------------------

1. Schema:

Create table: conversations
- id (uuid pk)
- taskId (uuid, nullable)
- participantOneId (uuid)
- participantTwoId (uuid)
- createdAt

Create table: messages
- id (uuid pk)
- conversationId (uuid)
- senderId (uuid)
- content (text)
- createdAt
- isRead (boolean default false)

Indexes:
- conversationId
- createdAt DESC

----------------------------------

2. Conversation Rules (IMPORTANT):

- A conversation can only exist if:
  - User is task creator OR accepted worker OR invited worker
- Prevent random spam:
  - No open chat between strangers without context

- Auto-create conversation:
  - When:
    - task is accepted
    - OR worker is invited
    - OR worker applies (optional future)

----------------------------------

3. API Routes:

POST /api/conversations/start
- Input: taskId, otherUserId
- Create or return existing conversation

GET /api/conversations
- Return all conversations for current user
- Include:
  - last message
  - unread count

GET /api/conversations/:id
- Return full message history

POST /api/messages
- Input: conversationId, content
- Save message
- Validate:
  - user is participant
  - content not empty
  - length limit (e.g. 2000 chars)

POST /api/messages/read
- Mark messages as read

----------------------------------

4. Safety Rules (CRITICAL — copy Fiverr-style logic):

- Block messages if:
  - user is not part of conversation
- Rate limit:
  - max 5 messages / 5 seconds (anti-spam)
- Basic content filter:
  - Flag messages containing:
    "WhatsApp", "Telegram", "call me", phone numbers
  - (warn user: "Keep communication on platform")

----------------------------------

Frontend

----------------------------------

1. Chat Entry Points:

- Task Detail Page:
  - "Message" button appears:
    - for creator ↔ worker
- Profile Page:
  - "Message" button (only if eligible)

----------------------------------

2. Chat UI Layout:

Left Sidebar:
- List of conversations
- Show:
  - name
  - last message preview
  - unread badge

Right Panel:
- Active chat

Message bubbles:
- Sent → right (purple accent)

----------------------------------

3. Message Features:

- Text only (MVP)
- Timestamp (e.g. "2:45 PM")
- Auto-scroll to latest

----------------------------------

- Show system messages:
  - "Task accepted — you can now chat"

  - task is closed AND no active dispute

  "Discuss details, clarify requirements, and share updates here"

5. Trust Enhancements:

  - name

----------------------------------

6. Empty States:

- No conversations:
  "No messages yet — start by accepting or posting a task"

----------------------------------

7. Notifications:

- Show unread badge in navbar

----------------------------------

Ensure:
