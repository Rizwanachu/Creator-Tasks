Build a full-stack application called "CreatorTasks" — an AI Content Job Board where users post AI content tasks (viral reels, hooks, thumbnails), others complete them for money, and the platform takes a 10% commission.

Tech Stack:
- Frontend: React + Vite + TypeScript + Tailwind CSS + shadcn/ui + Sonner (toasts)
- Backend: Node.js + Express + TypeScript
- Database: PostgreSQL (Neon free tier)
- ORM: Drizzle ORM
- Auth: Clerk (email + Google OAuth)

Core Rules:
- Budget is virtual escrow (no deduction on task creation)
- Commission is applied ONLY on approval (90% worker / 10% platform)
- All authenticated API requests must upsert the user via clerkId

Database Schema (STRICT — use exactly):

users:
- id (uuid, pk)
- clerkId (text, unique, not null)
- email (text)
- name (text)
- balance (int, default 0)
- pendingBalance (int, default 0)

tasks:
- id (uuid, pk)
- title (text)
- description (text)
- budget (int)
- status (text, default "open") // open, in_progress, submitted, completed
- creatorId (uuid)
- workerId (uuid, nullable)
- createdAt (timestamp)

submissions:
- id (uuid, pk)
- taskId (uuid)
- content (text)
- status (text, default "pending") // pending, approved, rejected

transactions:
- id (uuid, pk)
- userId (uuid)
- amount (int)
- type (text) // "earning" | "fee"
- createdAt (timestamp)

Backend Requirements:

1. Middleware:
- Authenticate via Clerk
- Extract clerkId
- Upsert user into DB on EVERY request

2. Routes:

POST /api/tasks
- Create task (title, description, budget)
- No balance deduction

GET /api/tasks
- Return all tasks with creator info

GET /api/tasks/:id
- Return task + submission + worker info

POST /api/tasks/:id/accept
- Atomic update:
  WHERE id = :taskId AND status = 'open'
- If 0 rows updated → return 409 "already taken"
- Set workerId + status = "in_progress"

POST /api/tasks/:id/submit
- Only worker can submit
- Create submission
- Set task.status = "submitted"

POST /api/tasks/:id/approve
- Only creator can approve
- Must be status = "submitted"
- Commission logic:

  workerEarning = floor(budget * 0.9)
  platformFee = budget - workerEarning

- Update worker balance:
  balance += workerEarning

- Insert transactions:
  (worker, earning)
  (creator, fee)

- Update task.status = "completed"

POST /api/tasks/:id/reject
- Only creator
- Reset:
  status = "open"
  workerId = null

GET /api/wallet
- Return balance, pendingBalance, transactions

GET /api/dashboard
- Return:
  - tasks created by user
  - tasks accepted by user
